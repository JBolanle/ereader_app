# Wrapup

End of session summary to maintain continuity.

## What to Produce

1. **List what was accomplished today**
   - Features implemented
   - Issues closed
   - Code written or modified

2. **Document any decisions made**
   - Technical choices
   - Architecture decisions
   - Patterns established

3. **Note any unresolved questions**
   - Things we weren't sure about
   - Questions to research later
   - Decisions that were deferred

4. **Suggest what to work on next**
   - Logical next steps
   - Open issues to pick up
   - Things to learn before proceeding

5. **Learning highlights**
   - New concepts understood
   - Skills practiced
   - "Aha" moments worth remembering

## Also Check

- Is CLAUDE.md still accurate? 
  - If not, show me the specific updates to make
- Should anything be added to the learning log?
- Are there any docs/architecture/ decisions to record?
- Is the "Current Sprint" section in CLAUDE.md up to date?

## Output Format

Format as markdown that can be pasted into `docs/sessions/YYYY-MM-DD.md`:

```markdown
# Session: [Date]

## Accomplished
- [What got done]

## Decisions Made
- [Decision]: [Reasoning]

## Open Questions
- [Question or uncertainty]

## Next Session
- [ ] [Task to pick up]
- [ ] [Thing to learn]

## Learned Today
- [Concept]: [Brief explanation of what I understood]

## CLAUDE.md Updates Needed
- [Section]: [Change to make]
(or "None - CLAUDE.md is current")
```
